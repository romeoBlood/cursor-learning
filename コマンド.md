# Cursor Commands（コマンド）

## 🧭 概要（30秒で思い出す）

- Cursor Commands は==「保存済みプロンプトのショートカット」==。定型作業を `/コマンド名` で即呼び出しできる
- `.cursor/commands/*.md` にマークダウンファイルとして配置するだけで使える
- ==引数は構造化されておらず、コマンド名の後に書いたテキストがそのままプロンプトに追加される==

## 🧩 要点まとめ

### コマンドの基本

| 項目 | 内容 |
|------|------|
| 配置場所 | `.cursor/commands/*.md` |
| ファイル形式 | プレーンなマークダウン（YAML frontmatter 不要） |
| 起動方法 | チャットで `/コマンド名` |
| 引数 | コマンド名の後に自由テキストを追記（後述） |
| 適用タイミング | ==手動起動のみ==（自動適用はされない） |

### 3 つのスコープ

| 種類 | 保存場所 | 用途 |
|------|---------|------|
| **Project Commands** | `.cursor/commands/` | プロジェクト固有の定型作業 |
| **Global Commands** | `~/.cursor/commands/` | 全プロジェクト共通の定型作業 |
| **Team Commands** | Cursor Dashboard | チーム共通（Team・Enterprise プランのみ） |

### 引数の仕組み

==コマンドには構造化された引数（`$1`、`{{name}}` 等）の仕組みはない。==
コマンド名の後に書いたテキストが、`.md` ファイルの内容と一緒にまるごとプロンプトとして AI に渡される。

```text
/commit and /pr these changes to address DX-523
```

↑ この場合 `and /pr these changes to address DX-523` が追加テキストとして `.cursor/commands/commit.md` の内容に付加される。

複数の情報を渡したい場合は、コマンドの `.md` ファイル内で「最初の単語を環境名、2番目をチケット番号として解釈せよ」のように AI への指示として書いておくのが現実的な回避策。

### 使い分け（Rules / Skills / Commands）

> [!tip] Zenn 記事からの引用
> - **Rules** は「常に守るべきルール」
> - **Skills** は「必要な時に呼ばれる専門知識」
> - **Commands** は「ユーザーが明示的に起動するショートカット」

| やりたいこと | 適切な仕組み |
|---|---|
| 毎回手動で呼び出したい定型プロンプト | **Commands**（`/コマンド名`） |
| タスクに関連する時に AI が自動で読み込んでほしい | Skills（段階的開示） |
| 常に適用したいルール・制約 | Rules（always / globs） |
| 全ファイルに共通の基本方針 | AGENTS.md |

> **ポイント**
> ==Commands は「ボタン一つでできる定型作業」。自動適用が必要なら Rules か Skills を使う。==

## 🧪 具体例・検証

### コードレビューコマンド

```markdown
<!-- .cursor/commands/code-review.md -->

# /code-review

## 手順
1. 変更されたファイルを確認
2. セキュリティ問題をチェック
3. コーディング規約に沿っているか確認
4. 改善提案を3つリストアップ
```

使い方: `/code-review この PR をレビューして`

### 公式ドキュメントに記載のあるコマンド例

- コードレビューチェックリスト
- セキュリティ監査
- 新機能セットアップ
- Pull Request 作成
- テスト実行と失敗修正
- 新規開発者オンボーディング

### Claude Code との比較

| 観点 | Cursor Commands | Claude Code Skills（コマンド的用途） |
|------|----------------|--------------------------------------|
| 本質 | 保存済みプロンプトのショートカット | ワークフロー定義（ツール・モデル・権限を含む） |
| 設定場所 | `.cursor/commands/*.md` | `.claude/skills/<name>/SKILL.md` |
| 起動方法 | `/コマンド名` | `/スキル名` |
| 引数 | コマンド名の後にテキスト追記（非構造化） | `$ARGUMENTS`（全体）/ `$0`（位置引数）で構造化 |
| 実行制御 | なし | ツール制限、モデル指定、サブエージェント指定 |

## ⚠️ つまずきポイント・アンチパターン

- ==**引数を `$1` や `{{name}}` で渡せると思い込む**== → 構造化引数はない。後続テキストがまるごとプロンプトに追加されるだけ
- **自動適用を期待して Commands に書く** → Commands は手動起動のみ。自動適用なら Rules か Skills を使う
- **すべての指示を Commands に集約する** → 常時適用すべきものは Rules / AGENTS.md に分離すべき
- **ファイル名を適当につける** → ファイル名がそのまま `/コマンド名` になるので、説明的な名前にする（例: `review-code.md`、`write-tests.md`）

---

## 🔁 Claude Code での活用方法

- Claude Code では Commands に相当する機能は Skills で実現する
- `/スキル名` で手動起動でき、`$ARGUMENTS` で引数を構造化できるため、より高機能
- 両ツール併用時は、Cursor 側で Commands、Claude Code 側で Skills と分けるよりも、`.claude/skills/` に統一する方が管理しやすい（Cursor も検出可能）

例：

```text
Claude Code で同等のことをする場合:
  .claude/skills/code-review/SKILL.md に定義
  → /code-review で起動
  → $ARGUMENTS でレビュー対象を受け取り可能
```
